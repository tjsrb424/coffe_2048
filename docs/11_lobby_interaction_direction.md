# 로비 인터랙션 방향 확정

## 결론
이 프로젝트의 로비는 **캐릭터를 방향키/가상패드로 직접 움직이는 구조를 사용하지 않는다.**
로비는 **고정 카메라 기반의 카페 화면 + 터치 중심 관리형 인터랙션**으로 설계한다.

즉, 레퍼런스 게임의 운영 루프는 참고하되, 조작 방식은 완전히 재해석한다.

---

## 왜 이동형 로비를 버리는가

### 1. 코어와 결이 맞지 않음
이 프로젝트의 메인 코어는 2048 퍼즐이다.
핵심 조작은 스와이프와 탭이며, 감정은 집중, 손맛, 몰랑한 반응, 힐링이다.

여기에 `Monkey Mart` 같은 직접 이동형 조작을 로비에 넣으면 아래 문제가 생긴다.
- 로비까지 노동형 플레이가 된다.
- 2048의 정제된 조작 감각과 충돌한다.
- 힐링보다 분주함이 앞선다.
- 모바일에서 조작 부담이 커진다.

### 2. 프로젝트 톤과 맞지 않음
이 게임은 귀엽고 정신없는 웹 캐주얼보다, 더 조용하고 부드럽고 세련된 톤을 지향한다.
따라서 로비도 '뛰어다니는 공간'이 아니라 '만지는 공간'이어야 한다.

### 3. 제작 난이도 대비 효율이 낮음
직접 이동형 로비는 아래 비용이 크다.
- 경로/충돌/상호작용 처리
- 애니메이션 상태 증가
- 모바일 조작 피로 증가
- 유지보수 비용 증가

반면, 고정 카메라 + 터치형 로비는 아래 장점이 있다.
- 구현 난이도 낮음
- Cursor로 작업 분할이 쉬움
- UI/모션 퀄리티를 높이기 쉬움
- 2048 코어와 자연스럽게 연결 가능

## 최종 로비 UX 원칙

### 기본 구조
- 카메라는 고정한다.
- 카페 전경 또는 카페 디오라마 형태로 보여준다.
- 캐릭터를 직접 움직이지 않는다.
- 핵심 상호작용은 탭 기반으로 진입한다.
- 일부 행동만 짧은 드래그/슬라이드 인터랙션을 허용한다.

### 로비는 완전 정적이지 않다
고정 카메라이지만 화면은 살아 있어야 한다.
예시:
- 커피 머신의 작은 불빛 반응
- 컵 위로 올라오는 김
- 손님이 앉아 있는 소형 루프 애니메이션
- 진열된 디저트의 작은 반짝임
- 팁 아이콘이 톡 뜨는 반응
- 시간대에 따라 조명 톤이 은은하게 달라지는 연출

즉, **조작은 정적이지만 시각 경험은 생동감 있게** 설계한다.

---

## 로비에서 허용하는 인터랙션

### 탭 중심 인터랙션
- 로스터 탭 → 원두 볶기 패널 오픈
- 쇼케이스 탭 → 메뉴 진열/재고 패널 오픈
- 테이블 탭 → 손님 만족도/팁/혼잡도 확인
- 카운터 탭 → 오늘 매출/주문 요약 확인
- 인테리어 탭 → 꾸미기/테마 변경
- 퍼즐 시작 CTA 탭 → 2048 화면 진입

### 제한적 드래그/슬라이드 인터랙션
- 완성된 메뉴를 쇼케이스에 짧게 드래그
- 원두를 로스터에 넣는 짧은 슬라이드 연출
- 팁 병을 탭/슬라이드해서 회수
- 특별 손님 주문 버블을 빠르게 탭해 보너스 획득

단, 이것은 장식적/감성적 상호작용이며, 로비 전체를 직접 컨트롤하는 조작으로 확장하지 않는다.

## 로비 플레이 루프

### 짧은 진입 루프
1. 로비 진입
2. 밤새 쌓인 판매 수익 확인
3. 팁/오프라인 수익 회수
4. 쇼케이스 재고 부족 확인
5. 원두/메뉴 생산 상태 확인
6. 2048 퍼즐 진입 또는 업그레이드 진입

### 중기 반복 루프
1. 퍼즐 플레이로 자원 획득
2. 자원으로 원두/메뉴/인테리어 성장
3. 손님 만족도 및 판매 효율 개선
4. 자동화 슬롯 또는 생산 효율 해금
5. 더 좋은 매장 상태 달성

---

## 화면 설계 원칙
- 한 손 조작 우선
- 핵심 CTA는 엄지 영역에 배치
- 화면의 대부분은 카페를 보여주는 감상 영역
- 세부 관리는 바텀시트/모달로 진입
- 작은 반응이 많아야 하지만 전체는 조용해야 한다
- 전면 팝업 남발 금지
- 정보 밀도보다 체감 품질 우선

---

## 구현시 주의사항
- 로비 오브젝트는 클릭 가능한 핫존 개념으로 설계한다.
- 핫존은 나중에 인테리어가 추가되어도 유지보수하기 쉽게 분리한다.
- 로비의 애니메이션은 상태 기반으로 설계한다.
  - 예: idle / highlighted / available / collecting / upgraded
- 2048 결과가 로비 특정 오브젝트와 연결되어 보이도록 피드백을 만든다.
  - 예: 퍼즐 종료 후 원두 획득 시 로스터 아이콘이 살아난다.
  - 예: 손님 러시 보상 획득 시 카운터에 반응 배지가 뜬다.

---

## 한 줄 정의
이 게임의 로비는 **Monkey Mart처럼 뛰어다니는 공간이 아니라, 살아있는 카페를 터치로 관리하는 고급스러운 감상형 운영 공간**이다.
