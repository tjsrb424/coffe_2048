# 레퍼런스 게임 매핑 문서

## 목적
레퍼런스 게임을 통째로 복사하지 않고, 각 게임에서 가져올 구조와 버릴 구조를 분리한다.
이 문서는 이후 사이드코어, 로비 UX, BM 설계의 기준으로 사용한다.

---

## 핵심 결론
이 프로젝트는 한 게임을 그대로 따르지 않는다.
아래처럼 **역할별 레퍼런스 조합형**으로 설계한다.

- 카페 감성 뼈대: `Cat Coffee Shop`
- 운영 회전 루프: `Monkey Mart`
- 메뉴/레시피/오프라인 경제: `Coffee Idle`, `Idle Coffee Business`
- 웹 장기 운영/BM 철학: `Farm RPG`, `Melvor Idle`
- 단순 코어의 장기 몰입: `Cookie Clicker`

---

## 1. Cat Coffee Shop에서 가져올 것

### 가져올 것
- 작은 카페에서 시작하는 성장감
- 카페라는 공간 자체가 주는 애착
- 테이블/머신/좌석/직원 확장 구조
- 손님이 오고 머물고 떠나는 카페 감성

### 가져오지 않을 것
- 직접 이동 중심 플레이
- 계속해서 테이블/머신 사이를 오가는 노동형 반복
- 지나치게 캐주얼하고 가벼운 시각 톤

### 우리 게임에서의 변환 방식
- 카페의 '성장 체감'은 가져온다.
- 조작은 직접 이동이 아니라 터치 중심 핫존 관리로 바꾼다.
- 그래픽은 더 조용하고 고급스럽게 재해석한다.

---

## 2. Monkey Mart에서 가져올 것

### 가져올 것
- 생산 → 진열 → 판매 → 수익 회수 → 확장 루프
- 매장이 점점 바쁘고 풍성해지는 구조
- 자동화/효율화의 중요성
- 신상품/신설비 해금으로 공간이 점차 복합화되는 구조

### 가져오지 않을 것
- 캐릭터를 계속 직접 이동시켜 일을 시작하는 구조
- 화면 전체가 항상 바쁘게 흔들리는 운영 리듬
- 빠른 템포 중심의 피로한 운영감

### 우리 게임에서의 변환 방식
- 운영 루프의 뼈대만 차용한다.
- 실제 실행은 자동 운영 + 부분 개입으로 구성한다.
- 유저는 관리자처럼 터치로 지시하고 수금/정리/보너스 처리를 한다.

---

## 3. Coffee Idle / Idle Coffee Business에서 가져올 것

### 가져올 것
- 원두/레시피 기반 성장
- 메뉴별 단가와 효율 차이
- 업그레이드 체감이 즉시 드러나는 경제 구조
- 오프라인 수익
- 자동화 매니저 개념

### 가져오지 않을 것
- 단순 숫자만 커지는 건조한 화면
- 감상 포인트 없는 클릭 위주 구성
- 너무 많은 팝업형 경제 UI

### 우리 게임에서의 변환 방식
- 메뉴 레시피와 원두 등급 개념을 만든다.
- 로스팅, 진열, 판매를 하나의 관리 루프로 묶는다.
- 수치 성장은 하되, 카페 화면에서 시각적으로 반영되게 만든다.

---

## 4. Farm RPG에서 가져올 것

### 가져올 것
- 광고에 기대지 않는 장기 운영 철학
- 잔잔한 일일 루틴 구조
- 꾸준히 들어오게 만드는 가벼운 목표 제공 방식
- 편안한 체류감을 주는 서비스 감성

### 가져오지 않을 것
- 텍스트 중심 구조 자체
- 대량의 메뉴 나열형 정보 설계
- 지나치게 시스템 위주의 인상

### 우리 게임에서의 변환 방식
- BM 철학과 일일 루틴 구조만 참고한다.
- '오늘 할 일', '주간 주문', '시즌 테마 목표' 같은 루프를 만든다.
- 광고는 보조 수단으로만 둔다.

---

## 5. Melvor Idle에서 가져올 것

### 가져올 것
- 장기 목표 트리
- 콘텐츠를 단계적으로 확장하는 구조
- 웹 기반 장기 플레이 설계 감각
- 무료 체험 후 확장형 BM 참고 가능성

### 가져오지 않을 것
- 복잡한 표/숫자 중심 시각 언어
- 장비/스탯 위주 관리 화면 결

### 우리 게임에서의 변환 방식
- 카페 성장 트리를 장기 목표 구조로 쌓는다.
- 예: 원두 연구 → 메뉴 연구 → 기계 연구 → 인테리어 연구
- 다만 시각 표현은 훨씬 감성적이고 촉감 있게 만든다.

---

## 6. Cookie Clicker에서 가져올 것

### 가져올 것
- 단순한 코어도 메타 설계가 받쳐주면 오래 간다는 원리
- 업그레이드, 해금, 성취, 누적 목표 설계
- 반복 행동이 점차 의미를 갖게 되는 구조

### 가져오지 않을 것
- 클릭 과부하
- 숫자 폭증 중심의 거친 인상
- UI가 전부 시스템처럼 보이는 설계

### 우리 게임에서의 변환 방식
- 2048은 단순하지만, 카페 메타가 깊이를 만든다.
- 퍼즐 점수는 곧 카페의 성장을 위한 자원이 된다.
- 반복 플레이가 새로운 설비/레시피/테마 해금으로 이어져야 한다.

---

## 사이드코어로 가져갈 실제 루프

### 세션 시작
- 로비 진입
- 오프라인 수익 회수
- 오늘의 주문/이벤트 확인
- 생산/진열 부족 상태 확인

### 운영 루프
- 원두 로스팅
- 메뉴 제작
- 진열
- 손님 자동 구매
- 팁/매출 회수
- 설비 업그레이드
- 자동화 강화

### 퍼즐 연결
- 2048 플레이
- 점수와 최고 타일에 따라 카페 자원 획득
- 특정 타일/콤보로 보너스 버프 획득
- 로비로 복귀해 성장 반영

### 장기 루프
- 새 메뉴 해금
- 새 설비 해금
- 새 카페 구역 오픈
- 인테리어 테마 수집
- 단골 손님/특별 손님 컬렉션
- 시즌 이벤트 진행

---

## 그래픽 방향 차별점
우리 게임은 레퍼런스보다 아래 방향으로 차별화한다.

### 레퍼런스 대비 더 강조할 것
- 고급스럽고 조용한 화면 구성
- 따뜻한 크림/우드/커피 컬러
- 부드럽고 몰랑한 전환
- 손끝 반응이 살아있는 UI
- 피로하지 않은 애니메이션

### 레퍼런스 대비 줄일 것
- 정신없는 화면 이동
- 플레이어 직접 이동 노동
- 장난감처럼 가벼운 템포
- 강한 네온/과한 이펙트

---

## 최종 정의
레퍼런스 게임들에서 가져올 것은 **운영 루프와 성장 구조**이고,
가져오지 않을 것은 **직접 이동형 조작과 분주한 캐주얼 템포**다.
