# Cursor 전달 업데이트 가이드

## 목적
최근 논의된 방향 변경 사항을 기존 기획서에 반영하고, 현재 진행 중인 프로젝트에 안전하게 이어붙이기 위한 전달 문서다.

---

## 지금 Cursor에게 바로 넘겨도 되는가
**기본 방향 문서로는 바로 넘겨도 된다.**
하지만 실제 구현 수정까지 맡길 때는 **현재 진행상황을 함께 주는 것이 훨씬 좋다.**

즉, 답은 다음과 같다.
- 문서 자체는 지금 바로 전달 가능
- 하지만 정확한 수정/이어작업을 위해서는 현재 상태 공유가 매우 권장됨

---

## 왜 현재 진행상황이 필요한가
현재 코드베이스가 이미 진행 중이라면, Cursor는 아래를 알아야 정확한 delta 작업이 가능하다.

### 필요한 이유
- 이미 만들어진 구조를 보존해야 한다.
- 기존 파일명을 기준으로 수정해야 한다.
- 이미 구현된 로비/퍼즐/UI 방향과 충돌하지 않아야 한다.
- 진행 중인 버그와 새 방향이 섞여 있을 수 있다.
- 같은 기능을 중복 구현하는 실수를 줄일 수 있다.

예를 들어:
- 로비가 이미 캐릭터 이동형으로 일부 구현되어 있다면, 수정 범위가 커진다.
- 퍼즐 화면과 로비 연결이 이미 진행된 상태라면 BM/사이드코어 연결 지점을 바꿔야 한다.
- 현재 오른쪽 스와이프 버그처럼 엔진 입력 이슈가 있으면 우선순위가 달라진다.

---

## Cursor에게 같이 주면 좋은 현재 진행상황
아래 정보가 있으면 좋다.

### 1. 현재 구현 완료 범위
예시:
- 로비 기본 화면 완료
- 퍼즐 보드 렌더 완료
- 점수 계산 완료
- 저장 로직 일부 완료
- 하단 탭바 완료

### 2. 현재 미완료 범위
예시:
- 로스터/진열/판매 루프 미구현
- 인테리어 시스템 미구현
- BM 미구현
- 오프라인 수익 미구현

### 3. 현재 문제점 / 버그
예시:
- 모바일에서 오른쪽 스와이프 미동작
- 퍼즐 결과 보상 반영이 약함
- 로비 UI가 너무 정적임
- 탭바 active state 미흡

### 4. 현재 수정한 파일 목록
예시:
- `src/features/puzzle2048/*`
- `src/app/lobby/page.tsx`
- `src/components/layout/*`
- `src/stores/*`

### 5. 이번에 건드릴 범위
예시:
- 기존 퍼즐 엔진은 유지
- 로비 상호작용 구조만 재정렬
- 문서 방향 반영 및 BM 구조 초안 추가

---

## 가장 좋은 전달 방식
아래 순서를 추천한다.

### 1단계: 방향 고정
Cursor에게 이 문서 세트를 먼저 읽게 한다.
특히 다음 문서를 우선순위로 읽힌다.
1. `11_lobby_interaction_direction.md`
2. `12_reference_mapping.md`
3. `13_web_bm_strategy.md`
4. `14_cursor_handoff_update.md`

### 2단계: 현재 상태 요약 전달
아래 템플릿으로 현재 상태를 함께 보낸다.

```md
현재 프로젝트 진행상황 요약:
- 완료된 것:
- 미완료된 것:
- 현재 가장 큰 문제:
- 이번 작업에서 건드려도 되는 범위:
- 건드리면 안 되는 범위:
- 참고해야 하는 핵심 파일:
```

### 3단계: 이번 작업 지시
그 다음 아래처럼 구체적으로 요청한다.

예시:
- 로비를 이동형이 아니라 고정 카메라 기반 핫존 구조로 재정리해줘.
- 기존 2048 퍼즐 코어는 유지하고 로비 상호작용만 재설계해줘.
- BM은 아직 결제 구현하지 말고 상품 구조와 UI placeholder만 잡아줘.
- 오른쪽 스와이프 버그 먼저 고쳐줘.

---

## 진행상황 없이 넘겨도 되는 경우
아래 상황이면 문서만 넘겨도 된다.
- 아직 코드 구현을 거의 시작하지 않았음
- 구조를 새로 잡는 단계임
- 기존 파일이 적고 리팩터링 비용이 낮음

## 진행상황이 꼭 필요한 경우
아래 상황이면 현재 상태를 꼭 같이 줘야 한다.
- 이미 여러 화면/스토어/훅이 만들어져 있음
- 퍼즐과 로비가 이미 연결돼 있음
- 버그 수정과 구조 변경이 동시에 필요함
- 기존 UI 방향을 살리면서 수정해야 함

---

## 바로 붙여넣을 Cursor 전달문

```md
최근 기획 방향이 조금 구체화되었다.
기존 문서에 아래 보완 사항을 반영해서 현재 프로젝트 구조를 점검해줘.

핵심 변경 사항:
1. 로비는 방향키/직접 이동형이 아니라 고정 카메라 기반 터치형 관리 구조로 간다.
2. 레퍼런스 게임에서는 운영 루프만 가져오고, 직접 이동형 노동 플레이는 가져오지 않는다.
3. BM은 웹 우선 기준으로 직접 결제형(테마팩/스타터팩/광고제거/패스) 중심으로 설계하고, 광고는 선택형 보조 수단으로만 둔다.
4. 2048 코어의 공정성을 해치는 과금은 금지한다.

먼저 아래 문서를 읽고 이해해줘.
- docs/11_lobby_interaction_direction.md
- docs/12_reference_mapping.md
- docs/13_web_bm_strategy.md
- docs/14_cursor_handoff_update.md

그 다음 현재 프로젝트 상태를 기준으로:
- 이미 구현된 것
- 새 방향과 충돌하는 것
- 유지할 것
- 리팩터링할 것
- 다음 작업 우선순위
를 정리해줘.

특히 현재 프로젝트는 2048 퍼즐 코어를 유지하면서, 로비와 사이드코어 방향을 이번 문서 기준으로 정리해야 한다.
작업 후 변경 파일 목록, 구현 요약, 리스크, 테스트 방법까지 보고해줘.
```

---

## 한 줄 결론
- 문서는 지금 바로 넘겨도 된다.
- 하지만 **현재 진행상황 요약 + 주요 파일 범위 + 현재 버그 목록**까지 같이 주면 훨씬 정확하게 진행된다.
